# 패키지 배포 및 버전 관리

2단계는 배포 체계를 구축하고, 버전 충돌을 방지하기 위한 의존성 관리 전략을 수립

---

## 문제 정의
사용자들이 내꺼 다운받아서 같이 설치되는 패키지들끼리 문제가 안생기게 하고싶다.

버전을 잘? 관리해서? 내 패키지? 버전을 사용자들에게 직관?적으로 보이게하고싶다?

버전에 대응되는 브랜치,깃태그? 관리전략.

---

## 성과 정의

* **외부 설치 가능성 확보**: NPM 레지스트리(또는 사설 레지스트리)를 통해 패키지를 성공적으로 배포하고 외부 프로젝트에서 `npm install` 성공.
* **시맨틱 버전 관리 체계 구축**: SemVer 규칙을 준수하여 패키지 변경 사항의 영향 범위를 사용자에게 명확히 전달.

---

## 수치화 지표

| 지표명 | 측정 방법 | 목표치 |
| :--- | :--- | :--- |
| 배포 성공률 | NPM 배포 명령어 실행 시 에러 없이 완료된 비율 | 100% |
| 외부 프로젝트 설치 성공률 | 외부 독립 프로젝트에서 설치 및 런타임 실행 성공 여부 | 전수 통과 |

## 상세 실행 전략

### 의존성 최적화

패키지가 외부에서 설치될 때 불필요한 무게를 줄이고 충돌을 방지합니다.

* **피어 디펜던시 관리**: 리액트와 같이 호스트 앱과 버전을 공유해야 하는 라이브러리를 `peerDependencies`에 정의합니다.
* **의존성 분리**: 실제 런타임에 필요한 것과 빌드 시에만 필요한 것을 `dependencies`와 `devDependencies`로 엄격히 구분합니다.

### 버전 및 락파일 제어

모노레포의 거대한 의존성 그래프를 안정적으로 유지합니다.

* **시맨틱 버전 적용**: 하위 호환성 여부에 따라 버전을 체계적으로 업데이트하는 프로세스를 도입합니다.
* **락파일 고정**: 배포 시점에 사용된 정확한 의존성 트리를 락파일(`package-lock.json` 또는 `yarn.lock`)을 통해 동료 개발자와 공유합니다.

### 배포 자동화 준비

외부로 나가는 최종 결과물을 검수합니다.

* **배포 파일 선별**: `package.json`의 `files` 필드를 사용하여 실제 실행에 필요한 파일만 배포 대상에 포함시킵니다.
* **배포 전 검증**: `npm publish --dry-run` 등을 통해 실제 배포 전 패키지 구성을 미리 확인합니다.
