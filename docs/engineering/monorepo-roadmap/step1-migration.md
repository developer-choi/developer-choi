# 워크스페이스 구축 및 전수 이관

소스코드 한곳으로 모아서 애플리케이션들이 동작을해야함.
패키지간 import 되야하고.
Client Component같은것도 되야하고.

스토리북 등 각 애플리케이션 스펙이 1단계에 다들어가야함.
프로젝트 폴더관리 규칙같은것도 여기서 정해져야함.

---

## 문제 정의

* **프로젝트 간 중복성**: 동일한 UI 컴포넌트나 비즈니스 로직이 여러 프로젝트에 흩어져 있어 유지보수 비용이 이중으로 발생함.
* **코드 불일치**: 각 프로젝트마다 조금씩 다른 코드를 사용하여 브랜드 일관성이 깨지고 버그 수정이 파편화됨.

---

## 성과 정의

* **관리 단일화**: 코드 수정 한 번으로 연결된 모든 프로젝트에 변경 사항이 반영되는 통합 관리 체계 구축.

## 상세 실행 전략

### 물리적 연결

가장 기초적인 npm 워크스페이스를 통해 폴더 간의 논리적 연결을 수행합니다.

* package.json의 workspaces 설정 방식과 심볼릭 링크, 하드링크

### 자산 전수 이관

한두 개가 아닌, 기존의 모든 페이지와 컴포넌트를 이관합니다.

* 상대 경로(../../)를 워크스페이스 스코프 경로(@repo/)로 일괄 전환하는 전략을 수립합니다.
* 기존 프로젝트의 모든 자원을 모노레포 폴더 구조에 맞게 재배치합니다.

### 실행 환경 검증

디자인시스템은 nextjs 환경에서 CC SC 동작잘해야함.
