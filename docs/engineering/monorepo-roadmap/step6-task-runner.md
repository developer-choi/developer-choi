# 타스크 러너 도입 및 타스크 파이프라인 최적화

5단계는 개별 패키지의 빌드 시스템(Vite) 구축이 완료된 후, 수많은 프로젝트 간의 실행 순서를 자동화하고 중복 작업을 제거하여 모노레포의 운영 효율을 극대화하는 단계입니다.

---

## 문제 정의

* **명령어 실행의 고통**: 패키지 개수가 늘어날수록 각 폴더로 이동하여 `npm run build`를 반복하는 것이 불가능해짐.
* **불분명한 빌드 순서**: UI 라이브러리가 빌드되지 않은 상태에서 앱을 실행하여 발생하는 참조 에러 및 의존성 꼬임 문제.
* **시간 낭비**: 코드 한 줄만 고쳤음에도 아무 관련 없는 모든 패키지를 처음부터 다시 빌드해야 하는 비효율성.

---

## 성과 정의

* **통합 타스크 제어**: 루트 경로에서 단 한 번의 명령으로 워크스페이스 내 모든 프로젝트의 빌드, 테스트, 린트를 동시에 제어함.
* **의존성 기반 자동 정렬**: 패키지 간의 참조 관계를 파악하여 가장 먼저 빌드되어야 할 패키지를 스스로 판단하고 순차 실행함.
* **캐싱을 통한 속도 혁신**: 변경된 파일이 없는 패키지는 이전 빌드 결과물을 즉시 복원(Full Turbo)하여 빌드 시간을 0초에 가깝게 단축함.

## 상세 실행 전략

### 타스크 파이프라인 설계

여러 Vite 프로젝트들이 유기적으로 돌아가도록 길을 닦습니다.

* **타스크 관계 설정**: `build` 타스크 실행 전, 의존하고 있는 패키지의 `build`가 반드시 선행되도록 `dependsOn` 설정을 구성합니다.
* **입출력물 정의**: 어떤 파일이 변했을 때(`inputs`) 어떤 결과물을 보존할지(`outputs`)를 명시하여 터보레포가 지능적으로 캐싱하게 합니다.

### 병렬 실행 및 리소스 활용

* **동시성 극대화**: 의존 관계가 없는 패키지들은 CPU 코어를 최대한 활용하여 병렬로 빌드하여 실행 시간을 최소화합니다.
* **선택적 실행**: `--filter` 옵션을 통해 현재 작업 중인 특정 앱과 그 앱에 필요한 라이브러리만 골라내어 실행합니다.

### 전수 검증 및 고도화

* **FULL TURBO 확인**: 모든 패키지를 한 번 빌드한 후, 아무것도 수정하지 않고 다시 빌드했을 때 수 초 내로 완료되는지 확인합니다.
* **로그 통합 관리**: 여러 패키지에서 쏟아지는 로그를 터보레포가 보기 좋게 정리하여 에러가 발생한 지점을 즉시 파악하게 합니다.

### 원격 캐싱 (Remote Caching)

Vercel·셀프 호스팅 등의 원격 캐시를 도입해 팀원·CI 간 빌드 결과물을 공유합니다.

* **affected 동작 원리**: `git`으로 현재 브랜치와 비교 대상 브랜치 사이 변경 파일 목록을 식별 → 모노레포 내 모든 패키지의 `package.json`을 분석한 의존성 그래프를 참조 → 변경 파일이 속한 패키지와 그 패키지에 의존하는 모든 하위 패키지를 'affected'로 지정.
* **결정성 확보**: `commit hash`, `build timestamp`처럼 빌드마다 결과가 달라지는 비결정적 요소가 캐시 키에 포함되지 않도록 스크립트를 작성합니다.
* **디버깅**: 캐시 문제 의심 시 `--force`로 강제 재빌드 후 결과를 비교 → Turborepo가 생성하는 빌드 로그에서 캐시 키 구성 요소를 분석 → Vercel 대시보드에서 원격 캐시 현황을 모니터링.
