## ai 코드 안전망 — 정적분석 + 2단계 훅

### 의도

ai가 작성하는 코드의 양이 늘수록 사람이 리뷰할 양도 늘어남. 리뷰어가 로직·설계에만 집중할 수 있도록, ai가 자주 만드는 실수는 커밋 전에 자동 차단되는 안전망을 구축.

### 개선 과정

- eslint 룰 직접 설정: ai가 자주 만드는 패턴(typescript **any** 등)을 프롬프트로 막아도 반복되는 걸 발견 → 룰로 박아 커밋 시점에 강제 차단
- 테스트 환경 구축: ai가 코드를 수정할 때마다 기존 동작이 깨지지 않았는지 자동 확인 — vitest + react testing library
- ai 코드리뷰 스킬 연계: 앞에서 못 잡는 문제는 코드리뷰 스킬이 담당, 반복 지적은 따로 축적하여 같은 지시를 두 번 반복하지 않음

### 배운 점

- ai 행동 교정은 "지시"가 아니라 "검사"로 풀어야 함. 자연어로 "하지마"를 반복해도 같은 실수가 또 나오지만, 린트 룰·타입·테스트로 박아두면 매번 다시 부탁할 필요 없이 자동 차단됨
- 안전망은 비용이 저렴한 단계부터 쌓아야 함. 정적분석(즉각·자동·비용 0) → 테스트(작성 비용 있지만 회귀 자동 검증) → ai 코드리뷰(토큰·시간 비용 큼) 순. 앞 단계에서 잡힐 문제를 비싼 단계까지 끌고 가지 않아야 사람의 리뷰 부담이 실제로 줄어듦
